# Database Overview

The IPAMT database is designed to support a modular, scalable, and multi-level
team + property + commission system.

## Core Principles
- Clean, normalized structure
- Minimal redundancy
- Clear foreign key relationships
- Support for multi-level hierarchies
- Future-ready for Property, Sales, and Commission modules

## Current Modules
- Authentication
- Team System

## Future Modules
- Property Management
- Sales & Booking
- Commission Engine
- Payout System
