# Module Structure

IPAMT is designed to support multiple modules:

## Current Modules
- Authentication
- Team System

## Future Modules
- Property Module
- Sales Module
- Commission Engine
- Payout System

## Module Folder Pattern
app/Modules/ Property/ Sales/ Commission/

Each module will contain:
- Controllers
- Services
- Repositories
- Models
- Routes
- Migrations

This ensures plug‑and‑play scalability.
