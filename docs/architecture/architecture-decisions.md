# Architecture Decision Records (ADR)

This file documents key architectural decisions.

---

## ✅ Decision 1: Use Service + Repository Pattern
**Reason:** Clean separation of concerns, scalable, testable.

---

## ✅ Decision 2: Use Sanctum for Authentication
**Reason:** Lightweight, secure, perfect for SPA + mobile apps.

---

## ✅ Decision 3: Use Invite-Based Team Hierarchy
**Reason:** Real-estate business model requires referral-based onboarding.

---

## ✅ Decision 4: Modular Future Expansion
**Reason:** Property, Sales, Commission modules will be added later.

---

## ✅ Decision 5: API Versioning
**Reason:** Allows backward compatibility for future updates.
